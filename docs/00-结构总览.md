# SonettoHere v2.0 — 项目结构总览

## 项目定位

SonettoHere 是一个基于 **LangChain + LangGraph** 构建的 **ReAct AI Agent** 教学项目。它以"温暖体贴的大姐姐 Sonetto"为人设，通过 30 个 Skill 覆盖任务管理、地图服务、网络查询、文件操作、开发工具等 10 个功能领域。项目同时支持 CLI 和 QQ Bot 双入口。

该项目的技术骨架直接可作为 LangChain ReAct Agent 的入门教案——代码量适中，架构分层清晰，每一层都对应 LangChain 生态中的一个核心概念。

---

## 技术栈

| 组件 | 技术选型 | 作用 |
|------|----------|------|
| Agent 框架 | `langgraph >= 0.2.0` | 构建 ReAct Agent 状态图 |
| LLM 后端 | `langchain-openai`（DeepSeek Chat） | OpenAI 兼容的 tool calling |
| 状态管理 | LangGraph `MemorySaver` + 自建 `ShortTermMemory` | 会话状态持久化 + token 裁剪 |
| 配置管理 | `pydantic-settings` | 从 `.env` 加载 API Key |
| 工具定义 | LangChain `BaseTool` + Pydantic `BaseModel` | 30 个 Skill 的参数校验与执行 |
| 终端输出 | `colorama` + LangChain `BaseCallbackHandler` | 彩色流式输出 |

## 运行时数据流

一次完整的对话回合经历了以下路径：

```
1. 用户输入 "明天北京天气怎么样？"
       │
2. CLI (_stream_events) 包装为 {"messages": [{"role":"user", "content":"..."}]}
       │
3. 注入增强上下文（长期记忆检索 + 偏好画像）
       │
4. LangGraph StateGraph 将消息送入 LLM
       │
5. LLM 决策 → tool_call: get_current_weather(city="北京", forecast=true)
       │
6. WeatherSkill._run() 执行 → 调用 UAPI → 返回天气数据 JSON
       │
7. LLM 接收 tool 响应 → 生成自然语言回答
       │
8. PrinterCallback 流式输出思考过程 + Skill 调用 + 最终回答
       │
9. 回合结束 → Extractor 提取摘要 → 写入长期记忆
```

## 关键设计决策

**1. 为什么用 `create_react_agent` 而非手动构建？**

LangGraph 的 `create_react_agent` 封装了 ReAct 循环的样板代码：LLM 调用 → 解析 tool_call → 执行工具 → 将结果送回 LLM → 循环直到 LLM 输出最终回答。手动实现需要处理消息合并、递归限制、异常恢复等细节。对于教学场景，使用预构建函数可以聚焦于理解 ReAct 原理而非工程细节。

**2. 为什么用 `get_doc=true` 两步调用模式？**

大多数 Skill 的第一个参数是 `get_doc: bool = False`。当 LLM 首次遇到某个 Skill 时，它先传 `get_doc=true` 获取同目录下 `SKILL.md` 的领域知识文档，再带真实参数执行。这样做的好处：将领域知识从代码中分离出来（Markdown 便于维护），同时避免了将全部 30 个 Skill 的文档塞入系统提示词（超出上下文限制）。

**3. 为什么自建记忆系统而不依赖 LangChain 内置方案？**

LangChain 提供了 `ConversationBufferMemory`、`ConversationSummaryMemory` 等内置方案，但本项目需要更精细的控制：基于 token 计数的自动裁剪（短期记忆）、关键词+置信度+时间衰减的多维度检索评分（长期记忆）、LLM 驱动的结构化信息提取。这些需求超出了内置方案的覆盖范围。

**4. 为什么用 Pydantic `BaseSettings` 管理配置？**

`BaseSettings` 自动从环境变量和 `.env` 文件加载配置，类型安全，支持默认值。相比 `os.getenv` 散落在各处，集中管理所有 API Key 更易于维护和审计。
